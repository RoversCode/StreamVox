# Qwen3 TTS Clone 系列能力说明

本文档面向希望接入 StreamVox 的客户，集中说明 `Qwen3 TTS Clone` 系列模型的具体能力、语言支持与版本差异。  
当前 StreamVox 已接入以下两个版本：

* `qwen3-tts-clone-0.6b-gguf`
* `qwen3-tts-clone-1.7b-gguf`

## 1. 支持语言

参考 Qwen3-TTS 官方公开说明，Qwen3-TTS 系列支持以下 10 种主语言：

* 中文
* 英文
* 日文
* 韩文
* 德文
* 法文
* 俄文
* 葡萄牙文
* 西班牙文
* 意大利文

## 3. 核心能力

### 3.1 单参考音色克隆

Qwen3 TTS Clone 系列支持基于**一段参考音频 + 对应参考文本**快速构建角色 Prompt。  
构建完成后，这个角色可以被重复复用，用来生成新的文本内容。

需要特别说明的是：  
在 Qwen3 TTS Clone 系列中，如果你已经明确知道目标生成文本的语言，建议**显式传入与推理文本一致的语言参数**，通常会比直接使用 `auto` 获得更稳定的发音、断句和整体表现。  
`auto` 更适合语言不确定、混合输入或快速试用场景；而在正式生成场景下，**语言已知时直接指定语言**通常效果更好。

示例：

参考音频文本：

```text
你好，欢迎使用 StreamVox，我们将为你提供稳定、自然、低延迟的语音服务。
```

目标生成文本：

```text
您好，您的订单已经发货，预计明天下午送达，请保持电话畅通。
```

推荐方式示例：

```text
参考音频语言：Chinese
目标生成语言：Chinese
language="Chinese"
```

不太推荐但可用的方式：

```text
参考音频语言：Chinese
目标生成语言：Chinese
language="auto"
```

### 3.2 文本理解与自然韵律

Qwen3 TTS Clone 系列并不是简单地逐字朗读。  
它对常见文本具备一定的语义理解能力，可以在常规情况下较自然地处理：

* 断句
* 停顿
* 轻微语气变化
* 陈述句、疑问句、感叹句的基本表达差异

示例：

```text
今天会议改到下午三点，别忘了提前十分钟到。
```

通常会读成较自然的通知口吻，而不是机械地逐字平推。

### 3.4 多语言生成

如果你的业务包含中英文混合或多地区版本，Qwen3 TTS Clone 系列可以作为统一底座来处理。  
例如：

```text
欢迎来到 StreamVox 控制台。Please verify your device before deployment.
```

在语言控制上，Qwen3 TTS Clone 系列有一个很实用的经验规则：

* 如果目标文本语言是明确的，例如整句中文、整句英文、整句日文，建议直接传对应语言，而不是使用 `auto`
* 如果输入内容本身就是混合语言，或者你不确定模型应该按哪种语言优先解释，再考虑使用 `auto`

示例：

```text
文本：Actually，今天下午三点我们再开一次会。
```

这类混合文本可以使用：

```text
language="auto"
```

而下面这类纯中文文本：

```text
今天下午三点我们再开一次会，请提前准备好演示材料。
```

更推荐直接使用：

```text
language="Chinese"
```

## 4. 两个版本的差异

### 4.1 Qwen3 TTS Clone 0.6B

`0.6B` 版本更偏**轻量与速度**，适合：

* 更关注响应速度
* 更关注资源占用
* 需要在较低硬件成本下部署
* 更偏向在线实时交互

### 4.2 Qwen3 TTS Clone 1.7B

`1.7B` 版本更偏**质量与自然度平衡**，适合：

* 对音色还原要求更高
* 对长句稳定性要求更高
* 对自然度更敏感
* 希望在延迟与效果之间取得更稳妥的平衡

### 4.3 选型建议

如果你更看重：

* **更低延迟、更低资源占用**：选 `qwen3-tts-clone-0.6b-gguf`
* **更好的自然度、更稳的克隆质量**：选 `qwen3-tts-clone-1.7b-gguf`
